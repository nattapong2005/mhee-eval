ข้อกำหนดของ UI
แต่ละหน้ามี บทบาทที่มีสิทธิ์ กำกับชัดเจน ห้ามระบบแสดงหน้าที่ไม่ใช่ Role ของผู้ที่ใช้งาน เช่น EVALUATEE จะเปิดหน้า ของ ADMIN ไม่ได้ หากพยายามเปิดให้ย้อนกลับไปหน้า /home
URL Path และหน้าที่ของแต่ละหน้า
/login เข้าสู่ระบบมีฟอร์ม Email, Password
/register หน้าลงทะเบียนผู้ใช้งาน ใช้ลงทะเบียนผู้ใช้ โดยสามารถลงทะเบียนเป็น EVALUATOR, EVALUATEE เท่านั้น ไม่อนุญาตให้ลงทะเบียนเป็นสิทธิ์ ADMIN กรอกข้อมูลชื่อ แผนก (เลือกจาก dropdown) อีเมล password และ role อย่าลืมตรวจสอบความถูกต้องของข้อมูลก่อนบันทึกด้วย
/home เป็นหน้าที่แสดงหลังจาก Login ให้แสดง Dashboard และ card menu ตามบทบาทผู้ใช้งานที่เข้าสู่ระบบ ได้แก่
1. Admin: แสดงการ์ดจำนวนการประเมินทั้งหมด, จำนวน EVALUATOR, จำนวน EVALUATEE เมื่อกดให้แสดงข้อมูลอันที่เลือก
-	จำนวนการประเมินทั้งหมด: ไปหน้า /admin/evaluations
-	จำนวน EVALUATOR: (ยังไม่ต้องทำลิงก์)
-	จำนวน EVALUATEE: (ยังไม่ต้องทำลิงก์)
2. Evaluator: แสดงจำนวนการประเมินที่ได้รับมอบหมาย เมื่อคลิกแล้วให้ไปหน้า /evaluator/evaluations 
3. Evaluatee: แสดงจำนวนรายการประเมินที่ได้รับมอบหมาย เมื่อคลิกแล้วให้ไปหน้า /evaluatee/evaluations 
/admin/evaluations จัดการข้อมูลการประเมิน
หน้าให้เพิ่ม/ลบ/แก้ไข (กำหนดวันเปิด-ปิดประเมิน) ประกอบด้วย 2 ส่วน
1. ตารางแสดงรายการประเมิน มีคอลัมน์ลำดับที่ ชื่อแบบประเมิน วันที่สร้าง วันเปิด วันปิดประเมิน ชื่อผู้สร้าง และปุ่มให้ดูรายละเอียด แก้ไข และลบการประเมิน เมื่อกดรายละเอียดจะไปหน้า /admin/evaluations/:id เมื่อแก้ไขให้แก้ไขได้เป็น inline form เมื่อลบให้แสดง confirm เพื่อยืนยันก่อนลบ
2. ปุ่มเพิ่มการประเมิน แสดงเป็น modal ให้กรอกชื่อแบบประเมิน กำหนดวันเริ่มและสิ้นสุดการประเมิน เมื่อกรอกเสร็จแล้วให้กดปุ่มบันทึก หรือกดยกเลิกเพื่อปิด modal หากกดบันทึกข้อมูลให้ update ข้อมูลรายการประเมินในตารางข้อ 1 ทันที
/admin/evaluations/:id รายละเอียดการประเมิน
แสดงรายละเอียดการประเมินตามที่เลือกจากหน้า /admin/evaluations ประกอบด้วย 3 แท็บคือ
1. แท็บหัวข้อประเมิน: แสดงหน้าให้กำหนดหัวข้อประเมิน ด้านบนสามารถเพิ่ม/ลบ/แก้ไขหัวข้อประเมิน (Topic) ด้านล่างให้แสดงรายการ Topic ที่สร้างตามลำดับ แต่ละ Topic ให้แสดงรายการตัวชี้วัด (indicator) เป็นตาราง ประกอบด้วย
- ลำดับ 
- ชื่อตัวชี้วัด 
- ประเภทตัวชี้วัด (SCALE_1_4 แสดงเป็น “ระดับ 1-4”,
  YES_NO แสดงเป็น  “ผ่าน/ไม่ผ่าน”) เป็น badge สีต่างกันให้สวยงาม และมี icon ประกอบ
- น้ำหนัก แสดงค่านำหนักของตัวชี้วัดข้อนั้น
- หลักฐานประกอบ ให้แสดงคำว่าต้องการ หรือไม่ต้องการ พร้อม icon ประกอบ 
- ปุ่มให้แก้ไข และลบตัวชี้วัด
- แถวสุดท้ายให้แสดงฟอร์มเพิ่มข้อมูลตัวชี้วัดแบบ inline โดยช่องประเภทตัวชี้วัดและตัวเลือกหลักฐานประกอบให้แสดงเป็น dropdown และมีปุ่มเพิ่มและยกเลิก เมื่อกดเพิ่มต้องตรวจสอบก่อนว่าน้ำหนักรวมทุกตัวชี้วัดต้องไม่เกิน 100% จึงจะให้เพิ่มข้อมูลได้ ถ้าเกินให้แสดงข้อความเตือนและไม่มีการเพิ่มข้อมูล พร้อมทั้งตรวจสอบความถูกต้องของข้อมูลที่กรอกด้วย เช่นห้ามเป็นค่าว่าง ห้ามผิดรูปแบบข้อมูล ฯลฯ ตามความเหมาะสม
2. แท็บจัดการคู่ประเมิน (Assignment): เป็นหน้าจับคู่ผู้ประเมิน กับผู้รับการประเมิน โดยฟอร์มให้กรอกข้อมูลผู้ประเมิน และผู้รับการประเมิน จาก dropdown จากนั้นกดปุ่มเพิ่มเพื่อบันทึกข้อมูล เมื่อบันทึกแล้วข้อมูลจะแสดงในตารางด้านล่าง แต่ละคู่ประเมินต้องมีปุ่มลบเพื่ออนุญาตให้ลบข้อมูลการจับคู่ประเมิน และต้องยืนยันข้อมูลก่อนลบได้
3. แท็บผลการประเมิน: แสดงผลการประเมินของคู่ประเมินแต่ละคู่ จาก dropdown เมื่อเลือกแล้วจะแสดงผลการประเมินด้านล่าง dropdown ประกอบด้วยชื่อแบบประเมิน ชื่อผู้ประเมิน ชื่อผู้รับการประเมิน และคะแนนรวมเป็น% จาก 100% โดยแสดงทั้งตัวเลข Progress Bar   และราย ละเอียดผลการประเมินโดยแบ่งตามหัวข้อประเมิน แต่ละหัวข้อแสดงตารางผลประเมินตามตัวชี้วัดประกอบด้วยคอลัมน์
- ลำดับที่ 
- ชื่อตัวชี้วัด
- ประเภทตัวชี้วัด  (SCALE_1_4 แสดงเป็น “ระดับ 1-4”,
  YES_NO แสดงเป็น  “ผ่าน/ไม่ผ่าน”) เป็น badge สีต่างกันให้สวยงาม และมี icon ประกอบ
- น้ำหนัก
- คะแนนที่ได้
- คะแนนที่ปรับแล้วเป็น %
- หลักฐานประกอบ (ถ้ามี) แสดงเป็นลิงก์เพื่อดาวน์โหลดเอกสาร
แถวสุดท้ายให้แสดงคะแนนรวมทั้งหมด นอกจากนี้แต่ละตารางตัวชี้วัด (แยกตามหัวข้อ) ต้องมีขนาดความกว้างคอลัมน์คงที่ทุกตาราง เพื่อให้แสดงผลสวยงามไม่โย้เย้ไปมา
/evaluator/evaluations รายการประเมิน
หน้ารายการประเมิน: แสดงรายการแบบประเมินที่ Admin มอบหมายให้ มีลักษณะเป็นตาราง ประกอบด้วยคอลัมน์ลำดับ ชื่อการประเมิน และคอลัมน์จัดการคู่ประเมินมีปุ่มจัดการคู่ประเมิน เมื่อคลิกที่ปุ่มให้ไปหน้า /evaluator/evaluations/:id 
/evaluator/evaluations/:id รายการคู่ประเมิน
หน้าแสดงรายการคู่ประเมิน: แสดงรายการคู่ประเมิน เพื่อบอกว่าแบบประเมินที่เลือก Evaluator ต้องประเมิน Evaluatee คนไหนบ้าง โดยส่วนบนให้แสดงชื่อการประเมิน, ช่วยเวลาประเมิน ด้านล่างให้มีรายชื่อผู้รับการประเมินที่ผู้ประเมินได้รับมอบหมาย เป็นตาราง ประกอบด้วยลำดับ ชื่อผู้รับการประเมิน สถานะตัวชี้วัด และสถานะการประเมิน
- ความคืบหน้า ให้แสดงจำนวนข้อที่ให้คะแนนแล้ว เทียบกับจำนวนตัวชี้วัดทั้งหมดพร้อม progress bar
- สถานะการประเมิน ถ้ายังไม่มีข้อมูลการประเมิน หรือประเมินยังไม่เสร็จสิ้น (ให้คะแนนทุกข้อ) ให้แสดงปุ่มเริ่มการประเมิน เพื่อไปหน้า /evaluator/assignment/:id ถ้าประเมินเสร็จแล้วให้ icon บอกว่าประเมินเสร็จแล้วพร้อมข้อความ (/ ประเมินเสร็จสิ้น) เมื่อคลิกแล้วจะไปเปิดหน้าผลการประเมิน /evaluator/assignment/:id/result
/evaluator/assignment/:id หน้าบันทึกข้อมูลประเมิน

หน้ารายละเอียดการประเมิน แสดงชื่อแบบประเมิน ระยะเวลาประเมิน ชื่อผู้ประเมิน และมีแท็บต่อไปนี้
1. รายละเอียดตัวชี้วัด แสดงตัวชี้วัดแยกตามหัวข้อ ถ้าข้อไหนจำเป็นต้องแนบหลักฐานให้แสดงช่องให้อัปโหลดไฟล์ ถ้าอัปโหลดแล้วให้แสดงลิงก์เพื่อเปิดดูเอกสารที่แนบแล้ว (ช่องอัปโหลดยังขึ้นอยู่เพื่อให้เปลี่ยนเอกสารแนบได้)
2. ผลการประเมิน ให้เปิดแท็บนี้ได้ก็ต่อเมื่อผู้ประเมินให้คะแนนครบทุกข้อแล้ว รายละเอียดให้แสดงดังนี้ ส่วนบนให้แสดงชื่อการประเมิน ผู้ประเมินต่อด้วยผลการประเมินเป็น % พร้อม Progress Bar ด้านล่างเป็นรายการตัวชี้วัดจัดกลุ่มตามหัวข้อ แต่ละหัวข้อแสดงชื่อหัวข้อประเมิน ต่อด้วยรายการตัวชี้วัดเป็นตาราง ประกอบด้วย
- ลำดับที่ 
- ชื่อตัวชี้วัด
- ประเภทตัวชี้วัด  (SCALE_1_4 แสดงเป็น “ระดับ 1-4”,
  YES_NO แสดงเป็น  “ผ่าน/ไม่ผ่าน”) เป็น badge สีต่างกันให้สวยงาม และมี icon ประกอบ
- น้ำหนัก
- คะแนนที่ได้
- คะแนนที่ปรับแล้วเป็น %
- หลักฐานประกอบ ถ้ามี แสดงลิงก์ดาวน์โหลดเอกสาร
แถวสุดท้ายให้แสดงคะแนนรวมทั้งหมด
นอกจากนี้แต่ละตารางตัวชี้วัด (แยกตามหัวข้อ) ต้องมีขนาดความกว้างคอลัมน์คงที่ทุกตาราง เพื่อให้แสดงผลสวยงามไม่โย้เย้ไปมา
